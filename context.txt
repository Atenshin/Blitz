# FRC Match Analysis System — Claude Code Project Specification
# Project Goal
Build a Python desktop application that analyzes FIRST Robotics Competition match videos.
The system should:
* Accept either:
  * a single YouTube video URL
  * OR a YouTube playlist URL
* Download the videos automatically
* Pull event and match metadata from The Blue Alliance API
* Detect robots and balls using YOLO
* Track shots and successful scores
* Attribute shots/scores to teams
* Segment stats by match period
* Export results to Excel
* Provide an interactive GUI with editable overlays and video playback controls
This project is intended for offline scouting/statistical analysis.
---
# Core Functional Requirements
## Video Input
Support:
### Single Video

python main.py --youtube "https://youtube.com/watch?v=..."
### Playlist

python main.py --playlist "https://youtube.com/playlist?list=..."
The application should:
* Download videos using yt-dlp
* Cache downloaded videos locally
* Normalize video format using FFmpeg
* Store videos under:

/videos/{event_key}/{match_key}.mp4
---
# TBA Integration
Use The Blue Alliance API.
API docs:
https://www.thebluealliance.com/apidocs
The system should:
* Pull all matches for an event
* Retrieve:
  * match keys
  * alliance team lists
  * match names
  * video metadata
Required endpoints:

GET /event/{event_key}/matches/simple
GET /match/{match_key}

# Match Association Logic
If playlist/video titles contain:
* q1
* qualification 12
* final 1 match 2
* etc.
then automatically associate videos with TBA matches.
If automatic association fails:
Provide a manual mapping UI.

# Detection System
Use:
* PyTorch
* Ultralytics YOLOv8 or YOLOv9
---
# YOLO Classes
The detector should identify:

robot_blue
robot_red
ball_ground
ball_airborne
goal
There is ONLY ONE scoring goal.
Do NOT implement:
* passes
* possession tracking
* carry tracking
* lower goal tracking
---
# Detection Architecture
YOLO should ONLY detect:
* robots
* balls
* goal
Do NOT attempt to classify:
* score events
* shots
* possession
Those should be inferred by logic later.
---
# Tracking System
Use:
* ByteTrack preferred
* DeepSORT acceptable
Track:
* robot identities
* ball trajectories
---
# Robot Identity Attribution
Each alliance contains 3 robots.
Use TBA alliance lists to map tracked robots.
Preferred implementation:
Use OCR on robot bumpers.
Suggested OCR libraries:
* PaddleOCR
* EasyOCR
Workflow:
1. Crop robot bounding box
2. Run OCR
3. Extract team number
4. Assign persistent team identity
Fallback:
Allow manual reassignment in GUI.
---
# Ball State System
Ball states:

GROUND
AIRBORNE
SCORED
UNKNOWN
---
# Ground Ball Detection
Conditions:
* low velocity
* near field floor region
* not intersecting robot box
Overlay color:
RED
---
# Shot Detection
A shot begins when:

ball velocity > threshold
AND
trajectory originates near robot
AND
ball enters airborne state
Overlay color:
GREEN
Increment:

shots_attempted += 1
---
# Score Detection
A shot counts as scored if:

ball trajectory intersects goal region
AND
ball disappears into goal
Increment:

shots_scored += 1
---
# Shot Attribution
Attribute shots to:

nearest_robot_to_ball_before_launch
Implementation:

if distance(ball, robot) < threshold:
    shooter = robot

# Field Calibration (Important)
The top camera introduces perspective distortion.
Implement field calibration using:

cv2.findHomography()
Transform:

pixel coordinates → field coordinates
This improves:
* trajectory prediction
* shot path accuracy
* robot proximity logic
* goal intersection logic
---
# Performance Strategy
DO NOT run YOLO during GUI playback.
Instead:
1. Preprocess detections
2. Cache all detections
3. Load cached detections during playback
Benefits:
* smooth scrubbing
* instant seeking
* responsive editing
* lower GPU usage
